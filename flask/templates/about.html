About Celara

What is Celara?

Think of Celara as a smart detective for finding planets around other stars. Just like how a detective looks for clues to solve a case, Celara looks for tiny patterns in starlight that might indicate a planet is passing in front of the star.

When a planet crosses in front of its star (like a mini-eclipse), it blocks a tiny amount of light - sometimes less than 1%! These dips in brightness create a pattern called a "light curve." Celara uses artificial intelligence to analyze thousands of these light curves and determine which ones are most likely caused by planets rather than other phenomena.

How Does It Work?

Step 1: Get the Data
You can upload data in three ways: a spreadsheet with planet measurements, raw telescope data files, or just type in a planet candidate number and Celara will download the data automatically from NASA.

Step 2: AI Analysis
Celara's "brain" (a neural network) examines the light patterns. It's been trained on thousands of examples of real planets and false alarms, so it learns to spot the difference.

Step 3: Results
You get a percentage score for each candidate - higher scores mean "more likely to be a planet." Celara also shows you graphs and statistics to help you understand what it found.

Advanced Technical Details

For Developers & Researchers: The following section contains technical implementation details.

Neural Network Architecture: Celara implements a multi-modal deep learning approach based on Google's AstroNet, combining three specialized branches:

  • Local View Branch: 1D ResNet processing 201-point folded light curves focused on transit windows
  • Global View Branch: 1D ResNet analyzing 2001-point full orbital period data
  • Auxiliary Branch: Dense layers processing stellar parameters (radius, temperature, SNR)

Model Specifications: The network uses approximately 71,000 parameters with Global Average Pooling for computational efficiency. Feature extraction creates a 2206-dimensional space combining temporal patterns and astrophysical constraints. Confidence scoring provides prediction reliability based on distance from the decision boundary Bayesian approximation.

Data Pipeline: Built with Python scientific stack including Lightkurve for NASA mission data handling, Astropy for coordinate transformations and FITS processing, and Pandas for tabular data manipulation. Supports Kepler/K2 and TESS light curves with automatic detrending and normalization.

Web Infrastructure: Flask backend with RESTful API design, Bootstrap 5 responsive frontend, WebSocket integration for real-time progress updates, and containerized deployment options. Includes automated testing pipeline and CI/CD integration.

Open Source: The complete source code, model architecture, and training notebooks are available on GitHub, following reproducible research practices and scientific transparency principles.
